


一个节点推荐10tb

一天一个亿数据，一年3.5tb



🍃 MongoDB
理论极限：无限（因为天然支持 Sharding）

实战极限：数百 TB 到 PB 级都有人在跑



🪟 SQL Server (MSSQL)
理论极限：524PB（Enterprise Edition 数据库大小上限）

实战极限：几十 TB 以内比较常见


🐘 PostgreSQL
理论极限：32TB（单表），数据库整体可达数百 TB

实战极限：几十 TB 以内


📦 各数据库的极限数据体积（理论 vs 实战）
🐬 MySQL（InnoDB）
理论极限：64TB 级别（单表）

实战极限：几 TB 就开始吃力



📊 总结表
数据库	理论极限	实战可行规模	分布式能力	适合场景
MySQL	~64TB 单表	1–10TB	弱（需业务分片）	OLTP
MSSQL	524PB	10–50TB	中等（企业版）	企业 OLTP/OLAP
MongoDB	无限（分片）	100TB–PB	强（原生分片）	文档/日志/大规模数据
PostgreSQL	32TB 单表	1–20TB	弱（需 Citus）	复杂查询、强一致性


🧠 工程师视角：真正的分界线
单机数据库（MySQL / PostgreSQL / MSSQL）  
→ 10TB 以上就开始痛苦

分布式数据库（MongoDB / TiDB / Cassandra）  
→ 100TB–PB 才是它们的主场
